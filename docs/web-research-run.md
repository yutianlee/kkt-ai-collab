# KKT Web Research Run Procedure

Use this procedure for high-quality manual web runs with ChatGPT and Gemini, or mixed runs where web agents are manual and API agents are called by the orchestrator.

## Required Web Modes

- ChatGPT: Extended Pro, or the strongest available long-reasoning mode.
- Gemini: Pro Deep Think, if available.

Use one persistent conversation per web AI for the whole run.

For A3/A4 API setup, see `docs/api-setup.md`.

## Research-Mode Rule

This is not a smoke test. Let each model reason carefully. For the KKT conjecture, prefer explicit hypotheses, exact constants, endpoint-variable checks, and gap detection over long but vague proof sketches.

All copied responses should be Markdown source. Use each web UI's own Copy response button when possible.

## Round Barrier

Do not start the next stage until every file required by the current stage has a real response.

For the default web-test config:

1. Stage A requires:
   - `handoff/<run-id>/round_XXX/responses/A1.md`, copied from ChatGPT Extended Pro
   - `handoff/<run-id>/round_XXX/responses/A2.md`, copied from Gemini Pro DeepThink
   - `rounds/<run-id>/round_XXX/responses/A3.md`, generated by the DeepSeek API
   - `rounds/<run-id>/round_XXX/responses/A4.md`, generated by the Qwen API
2. Stage B requires `handoff/<run-id>/round_XXX/reviews/A1.md`, `handoff/<run-id>/round_XXX/reviews/A2.md`, and the matching API-generated review files for A3/A4 under `rounds/`.
3. Stage C requires `handoff/<run-id>/round_XXX/judge/judge.md`, because A1 is the judge.

After each stage is filled, rerun the orchestrator command for that same round. It advances only when the barrier is satisfied.

## Commands

First, generate the A1/GPT bootstrap judge prompt for the legacy Round 11 material and the two latest strategies:

```powershell
python -m math_collab.bootstrap_judge --run-id kkt-main
```

Paste `rounds/kkt-main/round_011/prompts/bootstrap_judge.md` into ChatGPT Extended Pro. Save the copied answer to:

```text
handoff/kkt-main/round_011/judge/bootstrap_judge.md
```

Then rerun the same command to ingest it:

```powershell
python -m math_collab.bootstrap_judge --run-id kkt-main
```

Now generate or advance a KKT research-mode round, starting at the first normalized round after the bootstrap judge:

```powershell
python -m math_collab.orchestrator --config config/agents.web-test.json --problem problems/kkt_conjecture.md --run-id kkt-main --start-round 12 --rounds 1 --skip-missing-api
```

Or leave the watcher running while web models think:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\watch_web_research_run.ps1 -RunId kkt-main -StartRound 12 -Rounds 3 -PollSeconds 30
```

## Clipboard Helpers

Paste one prompt:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\paste_web_prompt.ps1 -PromptPath "rounds\kkt-main\round_012\prompts\A1_reasoning_12.md" -DelaySeconds 7
```

Save a copied web response:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\save_clipboard_response.ps1 -OutputPath "handoff\kkt-main\round_012\responses\A1.md"
```

Normalize copied Markdown after saving responses:

```powershell
python -m math_collab.normalize_markdown handoff/kkt-main/round_012/responses/A1.md
```

## Prompt Files

For each stage, paste the corresponding prompt file into the correct web conversation.

Reasoning:

```text
rounds/<run-id>/round_XXX/prompts/A1_reasoning_<round>.md
rounds/<run-id>/round_XXX/prompts/A2_reasoning_<round>.md
```

Review:

```text
rounds/<run-id>/round_XXX/prompts/A1_review_<round>.md
rounds/<run-id>/round_XXX/prompts/A2_review_<round>.md
```

Judge:

```text
rounds/<run-id>/round_XXX/prompts/judge_<round>.md
```

The default judge for `config/agents.web-test.json` is A1 ChatGPT Extended Pro.
